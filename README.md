### `scripts\apply_sync.py`

### Краткий обзор файла `apply_sync.py`

Скрипт предназначен для синхронизации данных между очередью заявок и MPP-файлами через COM-интерфейс MS Project.

---

### Назначение:
- Автоматизация обновления процента выполнения задач в MPP-файлах на основе данных из очереди.
  
---

### Фильтрация:
- По умолчанию обрабатываются все записи из очереди.
- Поддерживаются два типа фильтров:
  - `--project` — обработка заявок только для указанного проекта.
  - `--id` — обработка конкретной записи по её идентификатору (`ID`).

---

### Как обновляется MPP:
- Скрипт находит запись в MPP-файле по уникальному идентификатору задачи (`UniqueID`) и обновляет процент выполнения задачи указанным значением.
- Если задача не найдена или возникает ошибка при работе с COM, запись помечается как неудачная и сохраняется в очередь для дальнейшей обработки.

## 📌 Фильтрация в apply_sync.py

Скрипт `scripts/apply_sync.py` поддерживает выборочную обработку заявок:

```bash
# Обработать все заявки в очереди
python scripts/apply_sync.py

# Обработать заявки только для конкретного проекта
python scripts/apply_sync.py --project 149.xml

# Обработать только одну заявку по ID
python scripts/apply_sync.py --id 42

# Обработать проект по номеру (без .xml)
python scripts/apply_sync.py --project 149
```

## Запуск проекта Project Dashboard

# 1. Клонируем репозиторий
git clone https://github.com/DmitriyKabanov-AI/project-dashboard.git
cd project-dashboard

# 2. Создаём виртуальное окружение
python -m venv venv
venv\Scripts\activate

# 3. Устанавливаем зависимости
pip install -r requirements.txt

# 4. Копируем и заполняем .env
copy .env.example .env
# (редактируем .env — вписываем свои пароли)

# 5. Инициализируем БД (создаст admin/admin123)
python init_db.py

# 6. Запускает приложение
python app.py

### 7. Открыть в браузере
```
http://localhost:8080
```

### 8. Вход
- **Логин:** `admin`
- **Пароль:** `admin123`
(Или зарегестрировать другую учётную запись под свою роль)
